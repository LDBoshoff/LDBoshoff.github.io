Project Management System

Java Command-Line Project Management System

Overview:

The Java Command Line Project Management System is a robust and versatile program designed for an imaginary structural engineering firm. It was my first attempt at solving a real-world problem. This system simplifies project management tasks, allowing users to efficiently create, view, search, and update project details through a command-line interface. All project data is seamlessly stored in CSV files for easy access and retrieval. Despite its simplicity, it was a great learning oppurtunity and reinforeced my desire to continue exploring engineering solutions with code.

Key Features:

  • Command Line Interface: The system boasts an intuitive command-line interface that guides users through various operations. Users can effortlessly navigate the program and perform tasks with ease.
  • Project Creation: With the ability to create new projects, users can input project details, including project number, name, type, address, ERF number, cost, paid amount, deadline, and key personnel information (architect, contractor, customer).
  • Project Viewing: Users can view all projects, ongoing projects, completed projects, and overdue projects. This feature provides a quick overview of project status and helps prioritize tasks.
  • Project Search: The search functionality allows users to find specific projects using project numbers or names. This feature ensures quick access to project details, making it easier to manage and update project information.
  • Project Updates: Users can make real-time updates to project attributes, such as project number, name, type, address, ERF number, cost, paid amount, deadline, and key personnel information. Additionally, users can finalize projects, generate invoices, and mark projects as completed.
  • Data Persistence: All project data is stored in CSV files, ensuring data integrity and persistence across program sessions. Projects can be retrieved and managed seamlessly.
  • Error Handling: The system incorporates robust error handling, providing informative messages to users in case of incorrect inputs or unexpected errors.

Technologies Used:

  • Java
  • Object-oriented design principles for modular and maintainable code.
  • CSV file handling for data storage and retrieval.
  • Command-line interface for user interaction.

Benefits:

This project management system streamlines the workflow of a structural engineering firm, offering efficiency, organization, and ease of use. It empowers users to manage projects effectively, track progress, and generate invoices effortlessly. With its clean and intuitive design, the system is adaptable to various project management scenarios.

Critiques and Reflections:

While this project represents a significant achievement, I've identified areas where I would have done things differently or made improvements:

  • User Interface Enhancement: The command-line interface, while functional, could benefit from a more user-friendly and visually appealing design. Implementing a graphical user interface (GUI) could make the system even more accessible to users who may not be familiar with the command line.
  • Testing and Validation: In retrospect, I would have implemented more comprehensive testing and validation mechanisms to ensure data integrity and prevent input errors. Incorporating unit tests using JUnit and validation checks would enhance the system's robustness.
  • Documentation: Although the code is simple, additional documentation with JavaDoc, including comments, would make it easier for other developers to understand and contribute to the project.

Suggestions for Improvement:

In addition to my critiques, here are some suggestions for future improvements:

  • User Authentication: Implement user authentication to enhance security and restrict access to authorized users only. This could be crucial for real-world applications.
  • Database Integration: Consider transitioning from CSV storage to a relational database system for more efficient data management and scalability.
  • Data Backup and Recovery: Implement automatic data backup and recovery mechanisms to safeguard against data loss or corruption.
  • Design Pattern Integration: Explore the integration of design patterns like the Builder Pattern to enhance the construction of complex objects like Project instances. This refactoring can improve code readability and maintainability, especially as my project grows.

Conclusion:

The project showcases strong programming skills, including object-oriented design, user interface development, and data management. It's a valuable addition to any portfolio, demonstrating my ability to create practical and efficient software solutions for real-world scenarios, while also highlighting a commitment to ongoing improvement and learning.
