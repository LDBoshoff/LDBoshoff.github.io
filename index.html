Project Management System

Java Command-Line Project Management System

Overview:

The Project Management System is a powerful Java command-line application designed to streamline project management for a fictional structural engineering firm. This project marked my initial foray into real-world problem-solving, reinforcing my commitment to engineering solutions through code. The system simplifies project management tasks, providing a user-friendly interface for creating, viewing, searching, and updating project details.

Key Features:

Intuitive CLI:

The system boasts an intuitive command-line interface, enabling users to navigate effortlessly through a console menu and perform tasks with ease.

Project Creation:

Users can create new projects, inputting essential details such as project number, name, type, address, ERF number, cost, paid amount, deadline, and key personnel information (architect, contractor, customer).

Project Viewing:

Access an overview of all projects, including ongoing, completed, and overdue projects, aiding in task prioritization.

Project Search:

Quickly find specific projects using project numbers or names, facilitating efficient project management and updates.

Real-Time Updates:

Users can update project attributes in real-time, including project number, name, type, address, ERF number, cost, paid amount, deadline, and key personnel information. Additional functionalities include finalizing projects, generating invoices, and marking projects as completed.

Data Persistence:

The system ensures data integrity and persistence through CSV file storage, allowing seamless data retrieval and management across sessions.

Error Handling:

Robust error handling provides informative messages to guide users in case of incorrect inputs or unexpected errors.

Critiques and Reflections:

User Interface Enhancement:

Consider implementing a more user-friendly graphical user interface (GUI) alongside the CLI to cater to users less familiar with the command line.

Testing and Validation:

Future iterations could include comprehensive testing and validation mechanisms using tools like JUnit to enhance system robustness.

Documentation:

To facilitate collaboration, consider adding JavaDoc comments and comprehensive code documentation.

Suggestions for Improvement:

User Authentication:

Implement user authentication for enhanced security and access control.

Database Integration:

Transition from CSV storage to a relational database system for scalability and efficient data management.

Data Backup and Recovery:

Implement automatic data backup and recovery mechanisms to safeguard against data loss or corruption.

Design Pattern Integration:

Explore the integration of design patterns like the Builder Pattern to improve code readability and maintainability as the project grows.

Conclusion:

This project showcases my strong programming skills, emphasizing object-oriented design, user interface development, and data management. It serves as a valuable addition to my portfolio, demonstrating my ability to create practical and efficient software solutions for real-world scenarios while highlighting my commitment to continuous learning and improvement.
